# 批量重命名工具

> Windows 平台零依赖批量文件重命名 GUI 工具，双击即用。

基于 Windows 自带的 PowerShell + .NET WinForms 构建，无需安装 Python、Node.js 或任何第三方运行时。将文件夹放到本地任意位置，双击 `.bat` 即可启动图形界面。

---

## 功能特性

- **三种重命名模式**，每种都支持 **替换 / 删除 / 增加** 操作：
  | 模式 | 说明 |
  |------|------|
  | 关键字 | 按字面字符串查找替换、删除；或在指定位置插入文本 |
  | 指定位置字符 | 按起始位置 + 长度操作，支持**从前数**和**从后数** |
  | 格式匹配（通配符）| `*` 匹配任意多字符，`?` 匹配单个字符 |

- **递归子文件夹**：可选是否深入子文件夹，并自定义递归深度（0 = 无限制）
- **处理对象**：仅文件 / 仅文件夹 / 文件和文件夹
- **扩展名控制**：可选是否将文件扩展名纳入重命名范围
- **预览机制**：执行前预览全部新旧名称对比，显示冲突检测结果
- **操作日志**：每次执行生成带时间戳的 CSV 记录，详细记录原路径、新名称、状态、错误信息

## 快速开始

### 环境要求

- Windows 7 / 10 / 11（自带 PowerShell 5.0+ 和 .NET Framework）
- 无需额外安装任何软件

### 使用步骤

1. 将整个文件夹放到任意位置
2. 双击 `run.bat`
3. 在 GUI 中选择目标文件夹、设置重命名规则
4. 点击「预览」检查结果 → 确认无误后点击「执行重命名」

## 使用示例

### 示例 1：关键字替换

把文件名里的「草稿」全部替换为「正式」：

```
模式: 关键字 | 操作: 替换 | 查找: 草稿 | 替换为: 正式
```

### 示例 2：按位置删除字符（从后数）

删除文件名最后 4 个字符（比如去掉 `_old` 后缀）：

```
模式: 指定位置 | 操作: 删除 | 起始位置: 0 | 长度: 4 | 从: 后
```

### 示例 3：在文件名前插入前缀

在所有文件名最前面加上日期前缀 `20260522_`：

```
模式: 关键字 | 操作: 增加 | 插入位置: 0 | 从: 前 | 插入内容: 20260522_
```

### 示例 4：通配符模式删除前缀

删除所有 `IMG_xxxx_` 格式的前缀：

```
模式: 格式匹配 | 操作: 删除 | 通配符: IMG_*_
```

## 操作日志

每次执行重命名后，自动在 `logs/` 目录下生成 CSV 文件（UTF-8 with BOM，可直接用 Excel 打开），字段包括：

| 字段 | 说明 |
|------|------|
| 时间 | 操作执行时间 |
| 类型 | 文件 / 文件夹 |
| 原完整路径 | 重命名前的完整路径 |
| 原名称 | 重命名前的文件名 |
| 新名称 | 重命名后的文件名 |
| 新完整路径 | 重命名后的完整路径 |
| 状态 | 成功 / 失败 |
| 错误信息 | 失败时的错误描述 |

## 常见问题

### Q: 出现「禁止运行脚本」错误怎么办？

启动 `.bat` 已自动设置 `-ExecutionPolicy Bypass`，一般无需手动配置。  
如仍报错，可右键 `.bat` →「以管理员身份运行」。

### Q: 执行后发现改错了怎么办？

可查看 `logs/` 下对应的 CSV 日志，内含完整的新旧名称映射，可据此手动恢复。  
**建议**：重要数据操作前先点击「预览」确认，或提前备份。

### Q: 深度设置为 1 是什么意思？

深度表示向下进入子文件夹的层数：
- `0` = 无限制（递归所有层级）
- `1` = 仅当前文件夹 + 第一层子文件夹
- `2` = 当前 + 第一层 + 第二层子文件夹
- 以此类推

## 技术实现

- **语言**：PowerShell 5.1+
- **GUI 框架**：System.Windows.Forms（.NET WinForms）
- **编码处理**：脚本为 UTF-8，通过 `.bat` 包装以 `[System.IO.File]::ReadAllText` + UTF-8 显式读取，规避 PowerShell 默认 ANSI 编码问题
- **安全设计**：
  - 重命名前按路径深度倒序处理，避免父目录改名导致子路径失效
  - 执行前进行同批次重名检测、目标已存在检测、非法字符检测
